Generation 1 Pokémon types API

This is a simple API that returns the name and types of Pokémon as they appeared in Generation 1.

These are returned as a JSON object, with the form 

{ name: <Pokémon's Name>, types: [<Type 1>, <Possibly Type 2>] }

If the Pokémon only has one type, the types array will only have the one string.

USAGE:

  • api/name/<Name>
  • Get a Pokémon's types by name. If you input Bulbasaur, you get back Bulbasaur's name and type(s) in an array.
  • The name must be exact: If you want Mr. Mime, it has to be "Mr. Mime" exactly, not mr. mime, Mr Mime, or Mister Mime. If you want Farfetch'd, it has to be "Farfetch'd", not "Farfetchd", "farfetch'd", or "Farfetch’d".
  • If the name isn't an exact match, an error message will be returned.
  • api/dex/<Dex Number>
  • Get a Pokémon's name and types by Pokédex number. If you input 25, you get back Pokémon #25's name and type(s) in an array.
  • If an out-of-bounds number (or something else that isn't a number) is input, MissingNo. will be returned.

Q & A

  • Why only Generation 1? Are you some kind of elitist?
  • Information about the modern state of Pokémon is readily available from multiple sources (I personally prefer Bulbapedia, but Serebii is also a good option.) This was a bit more niche - and also a project thrown together over a weekend, so a Generation 1 API was faster and easier.
  • Why is MissingNo. the only visible glitch? Aren't there other glitch-mon?
  • Yes, there are others. MissingNo. is the most famous. In the future I may add a lookup by internal hex value, through which other glitch-mon will be viewable.
  • Why would someone even use this API? Wouldn't it be easier to use a static table or something?
  • The purpose of this API is to be a simple API, so that people who are learning web development have a front they can experiment with. If you want an API for this purpose that isn't Pokémon, I recommend consulting this public API list.
